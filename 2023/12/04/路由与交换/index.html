网络基础知识

一、IP地址

ipv4：

(1)、IP地址就是一个唯一标识，是一段网络编码。
(2)、IP地址由32位二进制数组成，为了方便记忆，将IP地址32位进制数进行分段，每段8位，共4段，然后将 没段8位二进制转换为十进制数，中间点号分割。这种表达方式称之为点分十进制。
(3)、IP地址的形式：X.X.X.X X的范围：0-255 #网络位(固定)+主机位(可分配)
255.255.255.255/24 32位 abc固定，d可变
a b c d
255.255.255.255/30 2个可分配主机IP地址；
第一个为网络地址（网关），最后一个为广播地址（广播），不变
cmd ipconfig #显示ip

ipv6：

(1)、IPv6 地址采用十六进制的表示方法，共 128 位，分 8 组表示，每组 16 位，每组表示 4 个十六进制数。
各组之间用 “ : ”号隔开，例如：1080:0:0:0:8:800:200C:417A
(2)、在 IPv6 地址段中有时会出现连续的机组0，为了简化书写，这些 0 可以用“ : : ”代替，但一个地址中只能 出现一次“ : : ”。
(3)、128 位的 IPv6 地址由 64 位网络地址和 64 位主机地址组成。其中，64 位的网络地址又分为 48 位的全 球网络标识符和 16 位的本地子网标识符

三、子网掩码

作用：

(1)、子网掩码必须结合IP地址一起对应使用。
(2)、只有通过子网掩码，才能表明一台主机所在的子网与其他子网的关系，使网络正常工作。
(3)、子网掩码和IP地址做“与”运算，分离出IP地址中的网络地址和主机地址，用于判断该IP地址是在本地网络 上，还是在远程网络网上。
(4)、子网掩码还用于将网络进一步划分为若干子网，以避免主机过多而拥堵或过少而IP浪费。

表示方法：

(1)、点分十进制不是发：二进制转换十进制，每8位用点号隔开
子网掩码二进制11111111.11111111.11111111.00000000，表示为255.255.255.0

(2)、CIDR斜线记法：IP地址/n
①、192.168.1.100/24，其子网掩码表示为255.255.255.0，二进制表示为11111111.11111111.11111111.00000000
②、172.16.198.12/20，其子网掩码表示为255.255.240.0，二进制表示为 11111111.11111111.11110000.00000000

分类：

①、缺省子网掩码，也叫默认子网掩码，即未划分子网，对应的网络号的位都置 1 ，主机号都置 0 。
未做子网划分的IP地址：网络号＋主机号

A类网络缺省子网掩码： 255.0.0.0，用CIDR表示为/8
B类网络缺省子网掩码： 255.255.0.0，用CIDR表示为/16
C类网络缺省子网掩码： 255.255.255.0，用CIDR表示为/24

②、自定义子网掩码，将一个网络划分子网后，把原本的主机号位置的一部分给了子网号，余下的才是给了子网的主机号。其形式如下：
子网划分后的IP地址：网络号＋子网号＋子网主机号
例如：192.168.1.100/25，其子网掩码表示：255.255.255.128

五、VMware网络连接模式

1.桥接模式：在桥接模式下，虚拟机的网络适配器被连接到主机物理网络的网卡上。这使得虚拟机能够直接与物理网络中的其他设备通信，并且虚拟机可以拥有与物理网络相同的 IP 段。虚拟机在桥接模式下可以获得独立的IP地址，就像它是网络中的另一台计算机一样。
2.NAT模式：在NAT模式下，虚拟机的网络流量通过主机操作系统进行网络地址转换。虚拟机的网络请求似乎来自主机，而不是直接来自网络。通常，主机操作系统充当了一个虚拟的路由器，为虚拟机提供了一种间接连接到物理网络的方法。
3.仅主机模式：在仅主机模式下，虚拟机之间和主机之间可以进行通信，但虚拟机无法与外部物理网络中的其他设备直接通信。这种模式提供了一种在虚拟化环境中创建独立、隔离的网络环境的方法，同时不让虚拟机访问外部网络。

每种模式都有其独特的用途。桥接模式允许虚拟机与外部网络直接交互，NAT模式提供了一种对外部网络进行隔离并允许虚拟机访问外部资源的方式，而仅主机模式则提供了一种独立、隔离的虚拟网络环境。
根据具体的需求和网络配置，你可以选择不同的网络连接模式来满足你的需求。

VMware虚拟网卡：在虚拟机环境中，虚拟化软件（VMware）会为每个虚拟机分配虚拟网卡，使虚拟机能够通过主机系统连接到网络。

路由与交换的实际应用

一、概念

路由器：路由器是一个网络设备，用于连接不同的网络，并在它们之间传递数据包。它具有多个网络接口，能够了解网络拓扑，并根据路由表中的信息选择最佳路径。
路由表：路由表是路由器上存储的信息表，其中包含网络的地址和路由器将数据包发送到这些网络所需的信息。路由表根据目标地址确定下一跳的路由器。
IP地址：IP地址是计算机网络中用于标识和定位设备的唯一地址。路由器使用IP地址来决定如何将数据包从源地址传递到目标地址。
静态路由：静态路由是由网络管理员手动配置的路由信息。在静态路由中，路由器的路由表不会自动更新，而是依赖管理员手动添加和维护。
动态路由：动态路由是一种自动更新路由表的方式，路由器能够通过网络协议（如OSPF、BGP、RIP）与其他路由器交换路由信息，从而动态地适应网络拓扑的变化。
子网掩码：子网掩码用于确定IP地址中哪部分是网络地址，哪部分是主机地址。它帮助路由器区分网络和主机，并在路由表中做出正确的路由决策。
默认网关：默认网关是一个路由器，用于处理目标地址不在本地网络上时的数据包。它是主机发送数据包到其他网络的出口点。

二、功能

静态路由的功能：
1.手动配置路径： 静态路由需要管理员手动配置路由表，明确指定数据包的下一跳路径。这使得网络管理员可以有针对性地控制数据包的流向。
2.简单和直观： 静态路由的配置相对简单，易于理解。网络管理员可以直接查看和编辑路由表，了解网络中每个节点的连接关系。
3.安全性： 由于静态路由不主动传播路由信息，因此在一定程度上可以提高网络的安全性。攻击者不容易获取关于网络拓扑的信息。
4.适用于小型网络： 在小型网络环境中，静态路由可以是一种有效的选择，因为网络拓扑变化相对较少，手动配置的开销相对较小。

动态路由的功能：
1.自动学习和适应： 动态路由协议可以自动学习网络拓扑信息，并根据网络的变化自适应地更新路由表。这使得网络更具弹性，能够适应拓扑变化。
2.负载均衡： 动态路由可以实现负载均衡，根据网络的状况选择最佳路径，分担流量，提高网络的性能和效率。
3.容错性： 动态路由协议能够在网络拓扑变化时重新计算路径，从而提高网络的容错性。当某个路径不可达时，动态路由可以选择备用路径。
4.可伸缩性： 动态路由适用于大规模网络，因为它可以自动适应网络规模的变化，并支持更复杂的网络拓扑。
5.自动更新： 动态路由协议能够自动传播路由信息到整个网络，减少了管理员手动维护路由表的工作，提高了网络的管理效率。

总体而言，静态路由适用于小型网络和需要手动控制路由的情境，而动态路由更适用于大规模网络和需要自动适应变化的环境，以提高网络的弹性和效率。选择使用静态路由还是动态路由通常取决于网络的规模、复杂性和管理需求。

(2).osi模型：

OSI模型（Open Systems Interconnection Model）是一个用于理解和描述计算机网络体系结构的框架。该模型将网络通信过程划分为七个不同的层次，每个层次都有特定的功能和责任。这种分层的方法有助于不同厂商的网络设备能够互相兼容，同时也使网络设计和维护更加模块化和灵活。以下是OSI模型的七个层次：

物理层（Physical Layer）：主要关注传输介质和物理连接，例如电缆、光纤、无线传输等。定义了数据传输的电气特性、物理连接和传输速率等。

数据链路层（Data Link Layer）：负责在直接连接的两个节点之间提供可靠的数据通信。提供了帧同步、流量控制和错误检测等功能。将物理层的比特流组织成帧（Frame）。

网络层（Network Layer）：主要负责数据包的路由和转发。提供了逻辑寻址、拥有路由器和交换机等设备。其中最著名的协议是IP（Internet Protocol）。

传输层（Transport Layer）：提供端到端的通信和数据传输。提供了差错检测、流量控制和数据重传等功能。有两个主要协议是TCP（Transmission Control Protocol）和UDP（User Datagram Protocol）。

会话层（Session Layer）：管理不同应用程序之间的对话。提供了会话的建立、维护和结束功能。

表示层（Presentation Layer）：负责数据的格式转换、加密和压缩等。提供了确保应用层能够解释收到的数据的手段。

应用层（Application Layer）：提供网络服务给用户和应用程序。包括网络服务协议，如HTTP、SMTP等。

这七个层次从物理层到应用层提供了一个逻辑上的网络框架，每一层都通过接口与相邻的层次通信。这种分层的设计使得不同层次的协议可以独立地设计、修改和扩展，同时也使得不同厂商的设备能够在同一网络中协同工作。

五、DNS

概念:DNS（Domain Name System）是一个用于将域名映射到IP地址的分布式命名系统。它充当了互联网的”电话簿”，通过将人类可读的域名转换为计算机可理解的IP地址，使用户能够更方便地访问互联网上的各类服务和资源。

域名结构：

①.域名是以层次结构组织的，由多个标签组成，标签之间用点号（.）分隔。

②.例如，”www.example.com“ 中，”www” 是子域，”example” 是二级域，”com” 是顶级域。

DNS解析过程：

①.当用户在浏览器中输入一个域名时，本地DNS解析器首先检查本地缓存，如果找到相应的IP地址，则直接返回结果。否则，它开始域名解析过程。

②.解析过程包括递归查询和迭代查询，涉及根DNS服务器、顶级域名服务器、权威域名服务器等。

DNS服务器类型：

①.递归DNS服务器： 大多数用户使用的DNS服务器，负责完成整个DNS解析过程。

②.权威DNS服务器：存储特定域名的DNS信息，提供域名解析的权威数据。

③.根DNS服务器：最高层次的DNS服务器，负责指导DNS解析的起始点。

六、网络拓扑搭建

网络拓扑是指计算机网络中各个网络节点之间的连接方式和结构布局。它描述了网络中设备之间的物理或逻辑关系，决定了数据在网络中如何传输和交换。

常见的网络拓扑包括：
1.星型拓扑（Star Topology）：所有的设备都连接到一个中心节点（如交换机或集线器），形成一个星型结构。这种拓扑方式简单、易于管理，并且在某个设备故障时不会影响整个网络。但如果中心节点故障，整个网络将无法正常工作。
2.总线型拓扑（Bus Topology）：所有的设备都通过一条共享的传输介质（如同轴电缆）连接在一起。设备之间共享传输介质，数据传递通过广播方式进行。这种拓扑方式使用方便，成本较低，但在传输介质上只能有一个设备传输数据，如果传输介质故障，整个网络将瘫痪。
3.环型拓扑（Ring Topology）：每个设备通过一个方向连接到相邻的设备，最后一个设备连接到第一个设备，形成一个环形结构。数据在环形结构中依次传递，直到到达目标设备。这种拓扑方式具有很好的数据传输效率，但如果一条连接故障，会导致整个环路中断。
4.树型拓扑（Tree Topology）：通过连接多个星型拓扑的中心节点来构建一个树状结构。这种拓扑方式可以扩展到更大规模的网络，允许分段和层级划分，但是依赖于根节点，如果根节点出现故障，则整个分支与根节点之间的通信将受到影响。
5.网状拓扑（Mesh Topology）：每个设备都与其他设备直接连接，形成一个网状结构。是一种高度冗余的拓扑方式，具有很好的容错性和可靠性，但是对于大规模网络，连接复杂度和成本较高。
